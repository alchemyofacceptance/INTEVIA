# Service Types v0.1

## Purpose

This file defines the categories of Services, their classification boundaries, and their relationship to the organism’s domains.

Service types answer:

> What kind of capability is this, and how should the organism govern its delivery?

In INTEVIA, Service type classification helps determine governance rules, provider requirements, evidence expectations, profile update boundaries, Event linkage, and downstream meaning.

## Boundary Note

For v1.0, Service types should remain minimal, explicit, and implementation-focused.

The priority is not to model every future capability category. The priority is to define enough classification structure for Services to be defined, assigned, delivered, evidenced, linked to Events, and governed clearly from the beginning.

Advanced semantic Service categories, automated Service interpretation, payment-linked Services, marketplace Services, full Exchange integration, and full Education integration remain deferred unless separately ratified.
