# Recognition Types v0.1

## Purpose

This file defines the categories of recognition artefacts and their classification boundaries.

Recognition types answer:

> What kind of becoming is this, and how should the organism govern its meaning?

Recognition types help the organism distinguish between different forms of evidenced becoming so that participation, contribution, capability, support, advocacy, and governance behaviour are recognised appropriately.

Recognition types apply to:

* learning;
* contribution;
* participation;
* engagement;
* governance behaviour;
* peer support;
* developer and volunteer contribution;
* external advocacy;
* field-signal contribution;
* capability formation.

## Boundary Note

For v1.0, Recognition types should remain minimal, explicit, and implementation-focused.

The priority is not to model every future form of recognition. The priority is to define enough classification structure for meaningful contribution, learning, support, advocacy, governance behaviour, and capability formation to be recognised without becoming status mechanics.

Badges, scoring categories, ranking categories, leaderboard categories, automated reputation categories, marketplace recognition types, payment-linked recognition types, and certification types remain deferred unless separately ratified.

Recognition types must classify evidenced becoming. They must not manufacture hierarchy.
