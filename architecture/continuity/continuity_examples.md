# Continuity Examples v0.1

## Purpose

This file provides safe, abstract examples of continuity movements without referencing real individuals, organisations, private data, or governed internal material.

Continuity Examples answer:

> What do continuity movements look like in practice?

These examples are illustrative only.

They show how continuity organs may interact without implying automation, prediction, hidden state, or real-world implementation maturity.

---

## Example 1: Drift Correction

### Situation

A thread contains two conflicting interpretations of a requirement.

### Movement

* drift is detected;
* correction is created;
* evidence is linked;
* decision is ratified;
* thread resumes.

### Organs Involved

* `continuity_corrections.md`
* `continuity_evidence.md`
* `continuity_decisions.md`
* `continuity_threads.md`

### What This Demonstrates

This example demonstrates how correction restores coherence before a workstream continues.

The key continuity movement is:

> pause → correct → ratify → resume

### What This Does Not Imply

This does not imply automatic drift detection, autonomous correction, or hidden memory reconstruction.
