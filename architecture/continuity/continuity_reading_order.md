
# Continuity Reading Order v0.1

## Purpose

This file defines the recommended conceptual reading order for the Continuity Engine v0.1.

Continuity Reading Order answers:

> How should a reader enter the Continuity Engine so the architecture makes sense?

This order is intended to support orientation, review, onboarding, and future implementation planning.

It is conceptual, not hierarchical.

---

## Alternative Entry Points

Although the sequence above is recommended, Continuity is non-linear.

A reader may also enter through:

* `continuity_reentry.md` when resuming after interruption;
* `continuity_risks.md` when diagnosing drift or fragmentation;
* `continuity_corrections.md` when coherence has broken;
* `continuity_threads.md` when managing parallel workstreams;
* `continuity_public_surface.md` when reviewing external claims;
* `continuity_lineage.md` when tracing how meaning moved across time.

Each entry point should still return to Purpose, Boundaries, Evidence, and Human governance before implementation decisions are made.
