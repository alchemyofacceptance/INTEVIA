# Library Resources v0.1

## Purpose

This file defines how Library artefacts are structured, versioned, linked, and governed.

A Library resource answers:

> What is this artefact, how is it structured, how is it versioned, how is it linked, and what evidence or lineage does it carry?

In INTEVIA, a Library resource is not a raw file. It is a governed knowledge object with type, structure, version history, lineage, evidence, and transition rules.

## Boundary Note

For v1.0, Library resources should remain minimal, explicit, and executable.

The priority is to support artefact definition, versioning, linkage, evidence capture, and governed state transitions.

Advanced knowledge engines, semantic resource graphs, automated inference, marketplace resources, payment-linked resources, full Exchange integration, and full Education integration remain deferred unless separately ratified.
