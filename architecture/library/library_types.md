# Library Types v0.1

## Purpose

This file defines the categories of Library artefacts, their classification boundaries, and their relationship to the organism’s domains.

Library types answer:

> What kind of knowledge or resource is this, and how should the organism govern its use?

In INTEVIA, Library type classification helps determine governance rules, versioning expectations, evidence requirements, resource linking behaviour, Service and Event linkage, profile update boundaries, and downstream meaning.

## Boundary Note

For v1.0, Library types should remain minimal, explicit, and implementation-focused.

The priority is not to model every future knowledge category. The priority is to define enough classification structure for Library artefacts to be defined, versioned, linked, evidenced, retrieved, and governed clearly from the beginning.

Advanced semantic Library categories, automated knowledge interpretation, payment-linked resources, marketplace resources, full Exchange integration, and full Education integration remain deferred unless separately ratified.
