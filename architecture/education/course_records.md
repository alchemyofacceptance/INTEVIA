# Course Records v0.1

## Purpose

This file defines how course records are structured, versioned, linked to curriculum, evidenced, and governed.

A course record answers:

> What is taught, how is it structured, how does it relate to curriculum and classes, and what evidence shows that it remains governed?

In INTEVIA, a course record is not a loose training page, content bundle, or informal lesson plan. It is a governed learning artefact with structure, versioning, lineage, evidence, curriculum linkage, class Event linkage, Library resource linkage, identity role boundaries, and transition rules.

## Boundary Note

For v1.0, course records should remain minimal, explicit, and executable.

The priority is to support course definition, versioning, curriculum linkage, class-as-Event linkage, Library resource linkage, learning evidence boundaries, and governed state transitions.

Full course authoring engines, semantic course graphs, adaptive learning paths, automated assessment mapping, certification engines, credential marketplaces, full Education 2.0 automation, and broad learner analytics remain deferred unless separately ratified.
