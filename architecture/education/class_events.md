# Class Events v0.1

## Purpose

This file defines how classes operate as governed Events, including attendance, delivery, evidence, and lineage.

A class event answers:

> When and how is learning delivered, who attends, what activity occurs, and what evidence is produced?

In INTEVIA, a class event is not merely a calendar entry, session listing, or attendance record. It is a governed learning Event with delivery context, learner and educator linkage, attendance rules, Service linkage, Library resource linkage, evidence capture, lineage, and transition rules.

## Boundary Note

For v1.0, class events should remain minimal, explicit, and executable.

The priority is to support class-as-Event definition, learner and educator linkage, attendance capture, delivery evidence, Library resource linkage, learning activity Service linkage, learning evidence boundaries, and governed state transitions.

Full cohort orchestration, automated assessment engines, adaptive classroom behaviour, live learning analytics, certification engines, credential marketplaces, full Education 2.0 automation, and broad LMS behaviour remain deferred unless separately ratified.
