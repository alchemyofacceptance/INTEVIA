# Education Overview v0.1

## Purpose

This file defines the purpose, scope, and conceptual stance of the Education Domain as the organism’s capability-formation substrate.

Education describes how curriculum, courses, and classes are defined, delivered, evidenced, and governed — and how learners acquire capability through governed Identity, Events, Services, Library resources, Governance Engine, Corpus, and CARE.

Education in INTEVIA is not a content library, not a CMS, and not a full LMS. It is a governed capability-formation substrate built on Identity, Events, Services, Library, Governance Engine, Corpus, and CARE.

## Boundary Note

For v1.0, Education should remain minimal, explicit, and executable.

The priority is to support curriculum definition, course definition, class-as-Event delivery, learner and educator linkage, learning evidence capture, Library resource linkage, and governed transitions.

Full LMS behaviour, automated assessment engines, semantic curriculum graphs, certification engines, credential marketplaces, cohort orchestration, adaptive learning engines, broad learner analytics, full Exchange integration, and Education 2.0 automation remain deferred unless separately ratified.
