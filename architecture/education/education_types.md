# Education Types v0.1

## Purpose

This file defines the categories of educational artefacts and their classification boundaries.

Education types answer:

> What kind of learning artefact is this, and how should the organism govern its use?

In INTEVIA, education type classification helps determine governance rules, evidence expectations, learner and educator role boundaries, Event linkage, Service linkage, Library resource linkage, and downstream capability formation.

## Boundary Note

For v1.0, education types should remain minimal, explicit, and implementation-focused.

The priority is not to model every future learning category. The priority is to define enough classification structure for curriculum, courses, classes, learning activities, and learning evidence to be linked, delivered, evidenced, and governed clearly from the beginning.

Advanced assessment types, certification engines, semantic curriculum categories, adaptive learning categories, marketplace education types, payment-linked education types, cohort orchestration types, full Exchange integration, and Education 2.0 automation remain deferred unless separately ratified.
