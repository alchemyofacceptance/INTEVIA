# Identity Records v0.1

## Purpose

This file defines how identity records are structured, linked, versioned, retrieved, and governed.

An identity record answers:

> Who is this actor, how is the record structured, how is it versioned, how is it linked, how can it be retrieved, and what evidence or lineage does it carry?

In INTEVIA, an identity record is not a user account. It is a governed actor object with type, structure, version history, lineage, evidence, retrieval boundaries, authority boundaries, responsibility boundaries, capability boundaries, and transition rules.

## Boundary Note

For v1.0, identity records should remain minimal, explicit, and executable.

The priority is to support identity definition, versioning, retrieval, linkage, evidence capture, authority attachment, responsibility attachment, capability boundaries, and governed state transitions.

Advanced identity engines, semantic identity graphs, automated inference, broad relationship modelling, marketplace identity, payment-linked identity, Exchange integration, and Education integration remain deferred unless separately ratified.
