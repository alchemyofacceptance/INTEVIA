# Event Lineage v0.1

## Purpose

This file defines how Events carry lineage, evidence, meaning, and historical continuity across the INTEVIA organism.

Event lineage answers:

> Where did this Event come from, what happened during it, what evidence does it carry, what changed because of it, and how is it remembered?

In INTEVIA, Events are not isolated records. They are experience objects that can connect people, services, resources, evidence, state transitions, audit events, and future meaning.

## Boundary Note

For v1.0, Event lineage should remain minimal, explicit, and executable.

The priority is not to implement a full lineage engine or semantic Event graph. The priority is to ensure that Events can carry enough history, evidence, audit linkage, state transition history, and resource relationships to remain trustworthy and meaningful from the beginning.

Advanced lineage engines, automated Event interpretation, semantic Event graphs, and broad historical visualisation remain deferred unless separately ratified.
