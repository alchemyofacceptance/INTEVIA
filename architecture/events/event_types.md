# Event Types v0.1

## Purpose

This file defines the categories of Events, their classification boundaries, and their relationship to the organism’s domains.

Event types answer:

> What kind of happening is this, and how should the organism treat it?

In INTEVIA, event type classification helps determine governance rules, state transitions, evidence expectations, attendance or participation behaviour, resource links, and downstream meaning.

## Boundary Note

For v1.0, Event types should remain minimal, explicit, and implementation-focused.

The priority is not to model every future Event category. The priority is to define enough classification structure for Events to move through governed lifecycles, carry evidence, link to resources, support participation, and remain understandable across the organism.

Advanced semantic Event categories, automated Event interpretation, event-driven automation, full notification systems, payment-linked Events, and marketplace Event types remain deferred unless separately ratified.
