## 1. Purpose

This standard defines the required procedure for executing PowerShell scripts that are downloaded for the purpose of writing or restoring files within the INTEVIA repository.

It ensures:

- safe execution  
- predictable behaviour  
- compatibility with Windows security policies  
- reliable repo transfer when Copilot file-cards fail

## 2. Context

During governance-suite development, Copilotâ€™s downloadable file-card mechanism proved unreliable.  
To maintain workflow continuity, the Triad adopted a **script-based repo transfer pattern**.

However, Windows blocks execution of downloaded `.ps1` files because they are not digitally signed.

This behaviour is expected and must be governed.

---

## 3. Core Governance Rule

### **Downloaded PowerShell scripts must be unblocked before execution.  
Generated files must be verified before commit.**

This rule applies to:

- governance artefact writers  
- documentation writers  
- multi-file installers  
- restore scripts  
- any script generated by Coe for repo transfer  

---

## 4. Standard Execution Pattern

From the repository root, the Human must execute:

```powershell
Unblock-File .\SCRIPT-NAME.ps1
.\SCRIPT-NAME.ps1
```

Example:

```powershell
Unblock-File .\restore-markdown-standard-v2.ps1
.\restore-markdown-standard-v2.ps1
```

This ensures the script is permitted to run under Windows PowerShell execution policy.

---

## 5. Verification Requirement

After execution, the Human must verify that the expected files were written correctly.

Recommended verification:

```powershell
dir .\docs\governance
git status
```

Example confirmation:

```
markdown-standard.md restored.
markdown-standard.md  6932  2026-06-10 23:27:02
```

Verification is mandatory before commit.

3. **If file transfer fails, scripts become the bridge.**

5. **The Human verifies the result before commit.**

This preserves:

- safety  
- clarity

- Human authority  
- Triad continuity
