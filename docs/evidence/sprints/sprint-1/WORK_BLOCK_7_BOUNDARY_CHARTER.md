# Work Block 7 Boundary Charter
*(Prepared boundary surface for Human review before Work Block 7 authorisation)*

## 1. Status of This Charter

This charter defines the constitutional perimeter for Work Block 7.

It is a boundary surface only.

It is not mutation authorisation.  
It is not an implementation plan.  
It is not final doctrine.  
It is not a feature commitment.  
It is not permission to expand without further Human authorisation.

Work Block 7 may proceed only after the Human Governor explicitly authorises a specific mutation surface.

---

## 2. Context

Sprint 1 has already established:

- a Sprint 1 evidence surface;
- a first runnable organism skeleton;
- governed status reporting;
- Human-readable runtime output;
- first external verification tests;
- study-event capture;
- the first named governed command surface: `heartbeat`;
- a Sprint 1 constitutional checkpoint;
- a bounded OPC doctrine note.

These artefacts establish the first rails for governed expansion.

Work Block 7 must operate inside those rails.

---

## 3. Constitutional Perimeter

Work Block 7 must preserve:

- Human authority;
- echo inspection;
- Git lineage;
- evidence-first progression;
- bounded claims;
- explicit non-claims;
- no silent state;
- no autonomous recovery;
- no expansion faster than evidence.

No mutation may proceed merely because it is technically convenient.

No command surface may be added without a paired evidence surface.

No behaviour may be introduced that the Human cannot inspect, explain, and verify.

---

## 4. Surfaces That Must Remain Stable

Work Block 7 must not break or obscure the meaning of:

- `breathe()` — first observable organism breath;
- `current_status()` — governed sprint context as data;
- `format_status()` — Human-readable governed status;
- `heartbeat()` — first named governed command surface;
- `run.py` — simple development entry point;
- Sprint 1 evidence log — bounded memory / study trail;
- Sprint 1 constitutional checkpoint — evidence rail;
- Sprint 1 OPC bounded doctrine note — doctrine-safe perimeter.

If any of these must change, a breaker must fire and the Human Governor must re-authorise.

---

## 5. What Work Block 7 May Explore

Work Block 7 may explore one clearly bounded next surface, such as:

- a new governed command surface;
- a status/inspection enhancement;
- a documentation index;
- a lightweight evidence helper;
- a boundary or routing scaffold.

The selected surface must be small, inspectable, testable, and reversible.

The preferred pattern remains:

> prepare → authorise → execute → echo → interpret → commit → remember

---

## 6. What Work Block 7 Must Not Do

Work Block 7 must not:

- introduce hidden state;
- introduce caching;
- introduce implicit memory;
- introduce autonomous mutation;
- introduce autonomous recovery;
- introduce a CLI framework without explicit authorisation;
- introduce dependencies without explicit dependency governance;
- collapse breath, status, heartbeat, and future commands into one indistinct surface;
- make claims of maturity, autonomy, self-governance, or architectural completion;
- proceed without echo inspection.

Expansion must remain subordinate to evidence.

---

## 7. Required Evidence for Work Block 7

Any authorised Work Block 7 mutation must capture:

- what changed;
- why it changed;
- what existing surface it touched or avoided;
- how it was externally verified;
- what test or echo confirmed;
- what claim the evidence supports;
- what claim remains unsupported;
- whether any breaker fired.

Evidence must be recorded before the block is treated as complete.

---

## 8. Breaker Conditions

A breaker must fire if:

- an unauthorised file changes;
- an expected echo differs from the plan;
- a test fails;
- dependency need appears unexpectedly;
- hidden state is introduced;
- Human comprehension becomes unclear;
- command meaning blurs with another surface;
- mutation begins to outrun evidence;
- a node attempts to convert interpretation into authority.

When a breaker fires:

1. halt mutation;
2. inspect state;
3. restore or correct only under Human authority;
4. record the event if it is evidence-bearing;
5. resume only after Human re-authorisation.

---

## 9. Work Block 7 Completion Condition

Work Block 7 may be considered complete only when:

- the authorised mutation is implemented;
- the Human has inspected the changed surface;
- external echo or tests verify expected behaviour;
- evidence is recorded;
- the change is committed with technical and mythic commit structure;
- the change is pushed to remote continuity substrate;
- working tree returns clean.

---

## 10. Keeper

> Work Block 7 does not open because momentum demands it.  
> It opens only if the rails can hold.

> Evidence must lead architecture, not the reverse.
