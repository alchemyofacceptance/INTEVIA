# GRP-13 Governed Amendment Cycle

Status: Governance cycle definition
Scope: GRP-13 constitutional artefacts
Boundary: v1.0-bounded
Operational effect: None
Runtime effect: None

## Purpose

This document defines the governed amendment cycle for GRP-13 constitutional artefacts.

Its purpose is to ensure that any future amendment to GRP-13 artefacts proceeds through explicit Human authorisation, bounded mutation, evidence review, commit verification, and lineage preservation.

This document does not amend any GRP-13 artefact.

It defines the cycle by which future amendments may be prepared, reviewed, and recorded.

## Amendment Cycle

A GRP-13 amendment must proceed through the following governed stages:

1. Amendment need identified.
2. Human Governor authorises amendment-cycle preparation.
3. Proposed amendment scope is defined.
4. Affected artefacts are identified.
5. Mutation boundary is declared.
6. Draft amendment text is prepared.
7. Human review is completed.
8. Repo mutation is performed only after explicit authorisation.
9. Diff is reviewed.
10. Commit is created with bounded technical message.
11. Push is verified.
12. Public or downstream use is authorised only if separately approved.

## Required Authorisation Phrase

The governing authorisation phrase for beginning this cycle is:

`I, Carmien Owen, authorise creation of: the governed amendment cycle for GRP-13 constitutional artefacts.`

This phrase authorises creation of the amendment-cycle governance document only.

It does not authorise a substantive amendment.

## Amendment Scope Boundary

Any future GRP-13 amendment must declare:

- affected artefact name;
- amendment purpose;
- mutation boundary;
- non-authorised scope;
- evidence source;
- review requirement;
- commit target;
- publication status.

No amendment may proceed from implication, conversational drift, or inferred consent.

## Non-Authorised Scope

This amendment cycle does not authorise:

- direct mutation of GRP-13 constitutional artefacts;
- public-surface publication;
- operational activation;
- runtime behaviour;
- validation claims;
- reinterpretation of ratified artefacts without explicit amendment authority;
- amendment bundling across unrelated artefacts;
- silent overwrite of constitutional lineage.

## Evidence Requirements

Each future amendment must preserve:

- pre-amendment state;
- amendment rationale;
- exact mutation scope;
- diff review;
- commit hash;
- push verification;
- post-push alignment;
- Human review marker.

Repo evidence remains the verification surface.

## Lineage Rule

A GRP-13 amendment must preserve lineage.

It may correct, clarify, extend, or supersede only within an explicitly authorised amendment boundary.

It may not erase the fact that a prior version existed.

## Human Governance Rule

The Human Governor remains the final authority for:

- whether an amendment is needed;
- whether a draft amendment is acceptable;
- whether mutation is authorised;
- whether commit and push are authorised;
- whether public-surface use is authorised.

## Boundary

This document may claim:

- a governed amendment cycle has been defined;
- future GRP-13 amendment work must proceed through explicit boundary and evidence controls;
- repo evidence remains the verification surface.

This document does not claim:

- GRP-13 has been amended;
- any constitutional artefact has been superseded;
- any public artefact is ready for publication;
- any validation claim has been established.

## Keeper

> Constitutional artefacts may evolve, but only through governed lineage.
